üñ•Ô∏è Arduino GIGA Display Overview

Throughout the project, the Arduino GIGA R1 WiFi board and its companion display have served as our main user interface. Our work focused on enabling the display of real-time glucose concentration values received via serial communication with a Python script. Key features of the final GIGA display implementation include:

  • Live concentration display: Values from CSV files are sent from Python to Arduino and rendered as on-screen readings.
  • Touchscreen interactivity: The UI includes user selection via checkboxes and the ability to add new users with an on-screen keyboard.
  • User system: A dynamic list of users can be selected before each reading. Selecting "New User" prompts an on-screen keyboard to register a new entry.
  • Reading system: The system waits for an incoming signal from Python before updating the concentration. Once received, the value is shown with appropriate formatting.
  • Display refresh and double-tap navigation: The user interface includes a built-in delay and logic to refresh the screen only when needed. Additionally, a double-tap gesture returns the user to the home screen or triggers other key UI transitions.
  • Concentration logging: Readings are tagged with user names and transmitted back to Python for structured storage in CSV format.
  • Improved font scaling and UI styling using LVGL for clarity and readability on the 800x480 display.
  • Integration with potentiostat functionality in Milestone 4 to support glucose readings derived from chronoamperometry.

This interface was iteratively developed through multiple milestones, progressively incorporating user input handling, automated display refreshes, and modular file organization across the Arduino and Python environments. It is now a fully functional frontend for our glucometer system.

To display a startup logo on the Arduino GIGA Display:

  1. Use a `.bin` file converted from a logo image (e.g., BinarySweeTech.bin).
  2. Store this binary in your Arduino sketch using the INCBIN macro. Add incbin.h to your folder if not already present.
  3. Create a header file like sweetech_logo.h that includes the binary or array data of the logo.
  4. Place all related files inside final_resources/logo/ within your Arduino folder structure.
  5. In your sketch, include: 
    #include "logo/sweetech_logo.h"
  6. Use LVGL or direct pixel rendering to show the logo on boot, such as: 
    tft.drawRGBBitmap(0, 0, sweetech_logo, 800, 480);

This logo appears on boot before transitioning to the interactive UI.
