Project Proposal: Potentiostat-Based Glucometer


Introduction

The increasing prevalence of diabetes necessitates the development of reliable, accessible, and accurate glucometers. Traditional glucometers utilize enzyme-modified electrodes for electrochemical glucose detection. This project aims to develop a low-cost, potentiostat-based glucometer prototype that employs cyclic voltammetry for glucose quantification. The system will be validated against standard glucose solutions to ensure accuracy and reproducibility, paving the way for further refinements in electrochemical biosensing technology.


Objectives

  • Construct a potentiostat capable of performing cyclic voltammetry.
  • Adapt the potentiostat for glucose detection using enzyme-modified electrodes.
  • Validate the prototype by calibrating it with standard glucose solutions.
  • Identify and address limitations to improve future iterations.


Materials and Components

Hardware:

  • IO Rodeostat ($240, already purchased)
  • Adapters for screen-printed electrodes (optional, depending on AimsStrip compatibility)
  • Power Supply (Compatible with Adafruit ItsyBitsy M4)
  • Battery Backpack (Adafruit LiIon/LiPoly Backpack for debugging and power switching)
  • Display Screen (Options: Arduino Display - $55 from eBay, Adafruit Display - $24, compatibility with IO Rodeostat for integration)
  • Microcontroller: Adafruit ItsyBitsy M4 ($15, consider purchasing three for redundancy)


Calibration and Testing:

  • Glucose test strips (AimsStrip, Amazon, $15.15 for 50 strips)
  • d-Glucose stock solution (250 g/L in PBS)
  • Phosphate-buffered saline (PBS) for dilutions
  • Calibration concentrations: 10–200 mg/dL


Benchmark fasting blood glucose ranges:

  • Normal: 70–100 mg/dL
  • Low: <70 mg/dL
  • High: >125 mg/dL


Software and Tools:

  • Arduino IDE for waveform generation and data acquisition
  • Processing script for real-time data visualization
  • Excel/Matlab for calibration curve fitting and data analysis


Methodology

  • Calibration:
    • Prepare glucose solutions (10–200 mg/dL) for calibration.
    • Conduct cyclic voltammetry scans, analyzing oxidation current peaks for correlation with glucose concentration.
    • Discard initial unstable scans and average subsequent scans for consistency.


  • Testing and Validation:
    • Evaluate performance using glucose standards within physiologically relevant ranges.
    • Compare prototype readings to known glucose concentrations to assess accuracy and linearity.
    • Identify potential anomalies such as saturation at high concentrations and address underlying causes (e.g., mixing issues, electrode kinetics limitations).


  • Data Analysis:
    • Generate calibration curves based on the Randles-Sevcik equation.
    • Analyze deviations at high glucose concentrations and adjust scan parameters accordingly.


Engineering Design Considerations

  • Signal Stability: Noise reduction through optimized resistor selection and scan rates.
  • User Interface: Develop an intuitive interface for user-friendly data retrieval and analysis.
  • Power Supply: Assess battery-operated vs. external power options for portable use.


Concepts for Future Improvements

  • Error Analysis: Improve accuracy by refining calibration and eliminating systematic biases.
  • Data Storage and User Profiles: Implement onboard memory to track readings over time.
  • Integration with Display Modules: Consider LCD or LED screens for real-time data visualization.


Minimum Viable Product (MVP)

  • Functional potentiostat using IO Rodeostat.
  • Basic glucose detection with standard calibration.
  • Python-based data output (no external display required).
  • Single-user operation without long-term data storage.
  • Acceptable margin of error (15%) per industry standards.


Main Goals

  • Develop a user-friendly glucose meter.
  • Achieve industry-standard error margins.
  • Optimize calibration to meet ISO specifications.
  • Store and analyze user data for long-term tracking.
  • Incorporate date/time functionality for better monitoring.



Engineering Design for a Glucose Meter

  • Design Considerations: Designing a glucose meter from scratch requires  attention to accuracy, durability, user-friendliness, and manufacturability. The device must reliably measure blood glucose levels while maintaining portability and ease of use for patients. Key design elements include:
    • Electrochemical Sensor Integration: Ensuring that the electrodes and sensor interface with the testing strip effectively.
    •  Microcontroller & Signal Processing: Selecting a suitable microcontroller to process sensor data and display readings accurately.
    • User Interface & Display: Designing an intuitive LCD or OLED display for easy readability of results.
    • Power Efficiency: Optimizing battery life to allow prolonged usage without frequent recharging.
    • Button and Screen Design: Ensuring that the buttons are tactile, responsive, and easy to press. The screen should have a backlight for readability in low-light conditions and use a high-contrast display to accommodate users with visual impairments
    • Test Strip Insertion Space: Designing a dedicated, easy-to-access slot for test strips that aligns properly with the sensor to ensure accurate readings. The space should be durable, resistant to contamination, and facilitate effortless insertion and removal of strips.


  • Material Selection:
    • Housing Material: The enclosure must be lightweight, durable, and resistant to chemical exposure. Common materials include:
      • ABS Plastic: Offers strength, impact resistance, and ease of manufacturing.
      • Polycarbonate (PC): Provides excellent optical clarity for display covers and high impact resistance.
      • Medical-Grade Silicone: Used for areas requiring flexibility and skin contact.
      • Sensor Material: Electrodes are typically made from gold, platinum, or carbon-based materials to ensure accurate glucose readings.
